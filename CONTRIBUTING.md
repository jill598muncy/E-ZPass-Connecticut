# Contributing to E-ZPass Connecticut


Thank you for your interest in contributing to **E-ZPass Connecticut**. We welcome improvements, bug fixes, documentation updates, and new features.


## Contribution Guidelines
Be respectful, provide clear explanations, and keep pull requests focused and small. Follow the project style and run tests before submitting.


## How to Contribute
1. Fork the repository.
2. Create a branch: `git checkout -b feature/your-feature`.
3. Make your changes.
4. Run tests and linters locally.
5. Commit with clear messages: `git commit -m "feat: add ..."`.
6. Push to your fork: `git push origin feature/your-feature`.
7. Open a Pull Request against `main`.


## Reporting Issues
Open an Issue in GitHub with:
- A descriptive title
- Steps to reproduce
- Expected vs actual behavior
- Screenshots or logs if applicable


## Suggesting Features
Use the Issues tab and label the issue `enhancement`. Describe the use-case, proposed UX, and any API changes.


## Code of Conduct
Treat everyone with respect. Harassment or hateful language will not be tolerated. Maintainers reserve the right to close issues or block accounts that violate this policy.


## Review Process
- Pull requests will be reviewed by at least one maintainer.
- Maintainable PRs should include tests or documentation changes when appropriate.
- The reviewer may request changes; please address comments and push updates.
- Once approved, a maintainer will merge the PR.


Thanks for helping to improve E-ZPass Connecticut!
